# 🎯 RD9 產品開發進度戰情室

以單一 HTML 檔案實作的產品／專案進度儀表板，架構仿照 RD3 的
[wise.icpdas.com/Schedule.html](https://wise.icpdas.com/Schedule.html?tab=WISE)：
**Google 試算表的每個工作表 = 網頁上的一個頁籤**，資料透過 Google Apps Script API 自動同步，無需後端伺服器。

---

## 📁 專案結構

```
.
├── index.html                     # 主程式（新架構，開啟即用）
├── google-apps-script.gs          # Google Apps Script 後端（部署到試算表）
├── RD9產品專案進度_新架構.xlsx      # 新架構試算表範本（含既有資料遷移結果）
├── index_Skeuomorphism.html       # 舊版儀表板（保留參考）
└── 產品開發進度追蹤_RD9.xlsx        # 舊版資料範本（保留參考）
```

## 🗂️ 頁籤與工作表設計

| 頁籤 | 工作表名稱 | 類型 | 說明 |
|------|-----------|------|------|
| UA 通訊服務器 | `UA` | 產品卡片 | UA／BRK／ECO 系列，卡片顯示「最新版本 → 預計版本」進度條 |
| UA I/O 模組 | `UA IO` | 產品卡片 | U-7500M 系列 OPC UA I/O 模組 |
| EZ-UAQ | `EZ-UAQ` | 時間軸 | 軟體版本歷程，同一產品的多列合併成直式時間軸 |
| 專案 | `專案` | 時間軸 | 依客戶分區塊，卡片顯示使用設備／是否客製／業務，階段紀錄為時間軸 |
| Service | `Service` | 時間軸 | 客戶技術服務歷程，同一「服務名稱」的多列合併為同一張卡片 |

### 兩種卡片類型

1. **產品卡片型（product）**：一列 = 一個產品。橫向進度條左節點是「最新版本」、右節點是「預計版本」，中間顯示預計版本完成百分比；點擊節點可切換下方顯示最新／預計版本內容。
2. **時間軸型（timeline）**：一列 = 一筆紀錄（版本／階段／服務紀錄）。相同「名稱」的多列會合併成一張卡片，由上（最新）到下（最舊）顯示直式時間軸，每筆紀錄可各自帶進度百分比。

## 📊 試算表欄位格式

### `UA`、`UA IO`（產品卡片型）

| 欄位 | 必填 | 說明 |
|------|:----:|------|
| 產品名稱 | ✅ | 產品型號，例：`UA-5231` |
| 產品種類 | ✅ | 網頁上的分區塊標題，例：`UA OPC UA 通訊服務器` |
| 平台 | | 選填，有值時顯示為標籤 |
| 產品圖片網址 | | HTTPS 圖片連結 |
| 產品描述 | | 卡片上的簡短說明 |
| 產品狀態 | ✅ | `開發中` / `更新中` / `發售` / `停產` / `洽詢` / `暫停出貨` |
| 維護者1～5 | | 格式：`中文名(英文名)`，例：`陳慶文(Sun)`，網頁顯示英文名標籤 |
| 最新版本編號／日期／內容 | | 內容一行一項，網頁轉條列，超過 15 行自動收合 |
| 預計版本編號／日期／內容 | | 同上 |
| 預計版本進度 | | `0`～`100` 整數；留空 = 不顯示百分比 |

### `EZ-UAQ`（版本時間軸型）

同上，但版本欄位為：`版本編號`、`版本日期`、`版本內容`、`版本進度`。
**一列 = 一個版本**，新版本往下加一列（試算表由上到下 = 由舊到新）。

### `專案`（時間軸＋專案資訊）

| 欄位 | 說明 |
|------|------|
| 專案名稱 | 同名多列合併為同一張卡片的多筆階段紀錄 |
| 客戶名稱 | 網頁上的分區塊標題 |
| 專案描述／專案狀態 | 狀態：`進行中` / `已完成` |
| 業務名稱／使用設備／是否客製 | 卡片資訊列（留空則不顯示該欄） |
| 維護者1～5 | 同產品表 |
| 階段名稱／階段日期／階段內容／階段進度 | 每列一筆階段紀錄，例：階段名稱填 `討論`、`報價`、`執行`… |

### `Service`（服務時間軸型）

| 欄位 | 說明 |
|------|------|
| 服務名稱 | 建議 `型號（客戶）`，同名多列合併成同一客戶的服務歷程 |
| 客戶名稱 | 分區塊標題（內部工作可填 `內部工作與 ECR`） |
| 服務描述／服務狀態 | 狀態：`進行中` / `已完成` |
| 相關產品 | 卡片資訊列，多個以 `、` 分隔 |
| 維護者1～5／階段名稱／階段日期／階段內容／階段進度 | 同專案表 |

> 名稱開頭為 `說明` 或 `_` 的工作表不會輸出到網頁，可放備註或下拉選單來源。

## 🚀 部署步驟

1. **建立試算表**：將 `RD9產品專案進度_新架構.xlsx` 上傳到 Google Drive，開啟後「檔案 → 另存為 Google 試算表」。
2. **部署 Apps Script**：試算表中「擴充功能 → Apps Script」，貼上 `google-apps-script.gs` 全部內容 → 部署 → 新增部署作業 → 網頁應用程式（執行身分：我；存取權：任何人）。
3. **設定網址**：複製部署網址，填入 `index.html` 中的 `APPS_SCRIPT_URL`。
4. **發佈**：將 `index.html` 推上 GitHub Pages 即完成。

### 新增頁籤

1. 在試算表新增工作表（依上方任一種欄位格式）。
2. 在 `index.html` 的 `TAB_CONFIG` 陣列加入一筆設定（工作表名稱、頁籤文字、`product` 或 `timeline` 類型、統計卡定義）。

### 網址參數（可分享特定檢視）

`index.html?tab=UA&search=5231&category=...&owner=...&status=發售`

## 📅 週報自動匯入（/weekly-import）

每週收齊週報後，用半自動流程把「本週有更新的內容」填入試算表。
設計原則：**程式做機械的事（轉檔、比對、備份、寫入），Claude 做理解的事（分析、歸類），人做最後把關（審核）**。

### 每週操作流程

1. 把當週週報放入 `weekly_reports/<週次>/`（例：`weekly_reports/2026_0629/`，檔名維持 `(週報)2026_0629_人名.doc` 格式）
2. 在 Claude Code 輸入：`/weekly-import 2026_0629`
3. Claude 自動執行：轉純文字 → 與上週週報 diff 找出新增內容 → 對照試算表現況分析歸類 → 產生更新提案（`proposal.json`）→ **輸出審核摘要給你確認**
4. 你審核（可要求修改），確認後 Claude 執行寫入：
   - 寫入前自動備份整份試算表到雲端硬碟「RD9戰情室備份」資料夾（保留最近 12 份）
   - 寫入結果登記到試算表的 `_匯入紀錄` 工作表，同一週不會重複匯入

### 初次設定（一次性）

1. 用最新的 `google-apps-script.gs` 更新 Apps Script 程式碼
2. 在 Apps Script 編輯器選擇函式 `setupToken` → 執行 → 從執行紀錄複製 token
3. 部署 → 管理部署作業 → 編輯 → 版本選「新版本」→ 部署
4. 複製 `tools/config.example.json` 為 `tools/config.json`，填入 token
5. ⚠️ `tools/config.json` 與 `weekly_reports/` 已列入 `.gitignore`，**不可推上公開 repo**

### 工具指令（也可手動執行）

```
python tools/weekly_import.py extract 2026_0629          # 週報轉純文字
python tools/weekly_import.py diff 2026_0622 2026_0629   # 產生每人本週差異
python tools/weekly_import.py fetch                      # 下載試算表現況
python tools/weekly_import.py backup                     # 手動備份
python tools/weekly_import.py apply weekly_reports/2026_0629/proposal.json   # 審核後寫入
```

### 資料還原（三道防線）

1. Google 試算表內建「檔案 → 版本紀錄」可回復任意時間點（最快）
2. 雲端硬碟「RD9戰情室備份」資料夾內有每次寫入前的完整副本
3. `_匯入紀錄` 工作表記錄每次匯入的時間、週次與對應備份檔名

## 🔍 頁面功能

- **統計卡**：依頁籤類型顯示（產品總數／開發中／更新中／發售，或專案總數／進行中／已完成）
- **搜尋 + 篩選**：名稱搜尋、種類（或客戶）、維護者、狀態，篩選條件會同步到網址
- **最後更新時間**：每個工作表獨立記錄（由 Apps Script 的 `onEdit` 觸發器自動更新）
- **版本內容收合**：超過 15 行自動收合，可展開
- **RWD**：行動裝置自動切換單欄與精簡時間軸版面

## 🛠️ 技術規格

- **前端**：純 HTML / CSS / Vanilla JavaScript（無框架、無外部 JS 函式庫）
- **後端**：Google Apps Script（`?sheet=all` 一次回傳所有工作表 + 各表最後更新時間）
- **字型**：Noto Sans TC、Space Mono（Google Fonts）
- **瀏覽器支援**：Chrome、Edge、Firefox、Safari（現代版本）
